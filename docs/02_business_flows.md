# 2. Бизнес-сценарии интеграции

## 2.1 Сценарий: Отправка проекта в ИПР

### Описание

Пользователь инициирует отправку проекта в инвестиционную программу (далее — ИПР) из ИС УИП в ИС КИПР.

### Предусловия

1. Проект не относится к типу «ТПиР»
2. Включена интеграция с ИС КИПР:

   * В разделе «Настройки → Общее → Интеграции» установлен чекбокс «Подключить интеграцию с системой»
3. В настройках интеграции указан корректный путь для сохранения файлов
4. Текущая версия проекта ранее не выгружалась

### Основной поток

1. Пользователь инициирует отправку проекта в ИПР
2. Система проверяет выполнение предусловий
3. Система формирует структуру хранения:

   * Папка с датой выгрузки
   * Подпапка с наименованием и версией проекта
4. Система формирует набор файлов по проекту:

   * Файлы формата Excel
   * Состав файлов определяется интеграционными потоками
5. Система сохраняет сформированные файлы
6. Система инициирует передачу данных в ИС КИПР

### Альтернативные потоки

#### A1. Интеграция отключена

* Отправка в ИПР недоступна
* Доступен альтернативный сценарий согласования без интеграции

#### A2. Некорректный путь для сохранения файлов

* При попытке отправки отображается сообщение:
  «Проверьте путь для сохранения в настройках интеграции»

#### A3. Проект уже выгружался ранее

* Повторная отправка проекта недоступна
* Доступен альтернативный сценарий согласования без интеграции

## 2.2 Сценарий: Массовая выгрузка проектов в ИПР

### Описание

Пользователь инициирует выгрузку группы проектов в рамках программы ИПР.

### Предусловия

1. Все проекты типа «ТПиР»:

   * имеют атрибут «Отправлен в ИПР»
   * атрибут получен из внешней системы
2. Проекты ранее не выгружались в ИПР
3. В настройках интеграции указан корректный путь для сохранения файлов

### Основной поток

1. Пользователь инициирует массовую выгрузку проектов
2. Система проверяет выполнение предусловий
3. Система формирует данные по проектам:

   * проекты типа «ТПиР»
   * проекты типа «Новое строительство» (если не выгружались ранее)
4. Для каждого проекта:

   * формируется структура хранения (папки)
   * формируется набор Excel-файлов
5. Система сохраняет сформированные файлы
6. Система инициирует передачу данных в ИС КИПР

### Альтернативные потоки

#### B1. Не выполнены предусловия

* Если хотя бы один проект не соответствует условиям:
  отображается сообщение:
  «Выгрузить проекты в ИПР можно только после отправки всех проектов типа ТПиР»
