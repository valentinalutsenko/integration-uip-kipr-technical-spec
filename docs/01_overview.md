# 1. Общая информация

## 1.1 Участники интеграции

* ИС УИП "Адепт" — мастер-система, источник данных
* ИС КИПР — система-приемник
* ЕИП — интеграционная платформа (транспорт и маршрутизация)

## 1.2 Назначение интеграции

Интеграция предназначена для передачи данных по инвестиционным проектам между системами с целью формирования и согласования инвестиционной программы.

Передаваемые сущности:

* Проект
* Паспорт проекта
* Контрольные точки

## 1.3 Общая архитектура

* Тип: асинхронная интеграция
* Транспорт: HTTP взаимодействие через ЕИП
* Формат данных: Excel-файлы

### Общий процесс

1. Пользователь инициирует отправку данных в ИС УИП
2. ИС УИП формирует Excel-файлы
3. ИС УИП отправляет данные в ЕИП через HTTP endpoint
4. ЕИП принимает сообщение и помещает его в очередь
5. ЕИП маршрутизирует сообщение в ИС КИПР
6. ИС КИПР обрабатывает данные

### Особенности

* Передача осуществляется асинхронно
* Excel-файлы выступают в роли payload
* Доставка сообщений обеспечивается механизмами ЕИП


## 1.4 Потоки данных

* Всего потоков: 19
* Каждый поток — отдельный Excel-файл

## 1.5 Требования ЕИП (кратко)

* Взаимодействие осуществляется через ЕИП
* Обеспечивается гарантированная доставка
* Используется централизованная маршрутизация
* Предусмотрен контроль статусов доставки


## 1.6 Ограничения и допущения

* Передача осуществляется только через файловый обмен
* Прямые интеграции между системами отсутствуют
* Формат файлов фиксирован (Excel)
